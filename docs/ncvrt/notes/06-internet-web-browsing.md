## 1. Introduction to the Internet

### What It Is

The **internet** is a global network of connected computers, mobile phones, servers, routers, data centers, and communication cables. It allows people and organizations to exchange information across the world.

### Why It Is Used

The internet is used for communication, education, banking, shopping, entertainment, government services, research, business, cloud storage, online classes, and artificial intelligence tools.

### Real-World Analogy

Think of the internet as a huge road network:

| Internet Concept | Road Analogy |
|---|---|
| Internet | Road network connecting cities |
| Website | Building or office |
| URL | Full address of the building |
| Server | Office that stores information |
| Client | Visitor requesting information |
| Router | Traffic junction |
| DNS | Phonebook/address directory |
| ISP | Road service provider |

### How the Internet Works

```text
Your Device -> Browser -> ISP -> DNS -> Web Server -> Website Response -> Browser Display
```

1. You type a website address.
2. Browser asks DNS to find the server IP address.
3. Request travels through your ISP and internet network.
4. Server sends back web page data.
5. Browser displays the page.

Open a browser and visit `https://www.digitalindia.gov.in`. Identify the protocol, domain name, and padlock icon.

## 2. Evolution of the Internet

### Web 1.0: Read-Only Web

Web 1.0 websites were mostly static. Users could read information but had limited interaction. Examples included early company pages, basic news sites, and static educational websites.

### Web 2.0: Social and User-Generated Web

Web 2.0 made the internet interactive. Users can create, upload, comment, share, like, and collaborate.

Examples:

- Facebook
- YouTube
- Wikipedia
- Instagram
- Blogs
- Online forums

### Web 3.0: Introduction

Web 3.0 is an emerging idea focused on decentralization, user ownership, blockchain-based identity, and peer-to-peer systems. It is still developing and should be understood as a concept, not as a fully mature replacement for the current internet.

### AI-Powered Internet Era

Modern internet usage now includes AI assistants and AI search tools. Instead of only opening links, students can ask questions and receive summaries, explanations, examples, and structured answers.

Examples:

- ChatGPT
- Google Gemini
- Microsoft Copilot
- Claude
- Perplexity AI

### Important Lesson

Internet usage has evolved from reading pages, to creating content, to asking AI-powered systems for direct answers. However, students must verify important information from trusted sources.

## 3. Internet Fundamentals

### Internet

A worldwide network that connects devices and allows communication and data sharing.

### WWW

**WWW (World Wide Web)** is a system of linked web pages accessed using browsers. The web is one service on the internet. Email, file transfer, video calls, and online gaming are also internet services.

### Website

A collection of related web pages under one domain name. Example: `digitalindia.gov.in`.

### Web Page

A single page displayed in a browser. Example: a login page, article page, or contact page.

### Domain Name

Human-friendly website name such as `google.com` or `uidai.gov.in`.

### Hosting

Hosting means storing website files on a server so people can access them through the internet.

### DNS

**DNS (Domain Name System)** converts domain names into IP addresses. It works like a phonebook for the internet.

### IP Address

An IP address is a numerical address of a device or server on a network, like a house number.

### Server

A server stores websites, files, databases, videos, or applications and sends them to users when requested.

### Client

A client is the user's device or software requesting information, such as your phone, laptop, or browser.

### Protocol

A protocol is a set of rules for communication. HTTP and HTTPS are web protocols.

### HTTP

**HTTP (HyperText Transfer Protocol)** transfers web pages but is not encrypted.

### HTTPS

**HTTPS (HTTP Secure)** encrypts communication between browser and website. It is essential for login, banking, payments, and forms.

### URL Structure

```text
https://www.example.gov.in:443/services/index.html?page=1#top
|----|   |-----------------| |--| |----------------| |----| |---|
protocol       domain       port       path          query fragment
```

### HTTP vs HTTPS

| Feature | HTTP | HTTPS |
|---|---|---|
| Security | Not encrypted | Encrypted |
| Padlock | Usually absent | Usually present |
| Use | General reading only | Login, payments, forms |
| Risk | Data can be intercepted | Safer for sensitive data |

## 4. Internet Connection and ISP

### ISP

An **Internet Service Provider** gives internet access. Examples: Jio, Airtel, BSNL, local broadband providers.

### Connection Types

| Type | Use |
|---|---|
| Fibre broadband | Home, school, office, high-speed internet |
| Mobile data | Phone internet using SIM |
| Wi-Fi | Wireless local access through router |
| Hotspot | Sharing mobile internet to laptop |
| Public Wi-Fi | Railway station, library, cafe, campus |

### Data Terms

- **Bandwidth:** data capacity per second, often Mbps.
- **Data limit:** total data allowed in GB.
- **Latency:** delay in response, important for video calls and gaming.
- **Signal strength:** quality of Wi-Fi/mobile connection.

### Safe Connection Practice

Avoid banking, UPI, or password entry on unknown public Wi-Fi. Use mobile data or trusted networks for sensitive work.

## 7. Search Engines

### Traditional Search Engines

- Google
- Bing
- Yahoo
- DuckDuckGo

### Crawling

Search engines use automated programs called crawlers to discover web pages.

### Indexing

Discovered pages are stored in a large searchable index.

### Ranking

Search engines rank pages based on relevance, quality, freshness, popularity, location, and many other signals.

### Search Results

Search results include links, snippets, images, videos, maps, ads, and sometimes direct answers.

### Effective Search Techniques

| Technique | Example |
|---|---|
| Specific keywords | `NIELIT CCC syllabus 2026` |
| Exact phrase | `\"Digital India mission\"` |
| Government sites | `site:gov.in DigiLocker` |
| PDF files | `filetype:pdf CCC study material` |
| Exclude word | `python tutorial -snake` |
| Define term | `define browser` |

### Evaluating Search Results

- Check date.
- Check author or organization.
- Prefer official domains such as `.gov.in`, `.edu`, `.nic.in`.
- Avoid clickbait.
- Cross-check important facts.

|---|---|---|

## 10. Downloads, Uploads, Cookies, Cache, and Privacy

### Download

Downloading means saving a file from the internet to your device.

Examples:

- PDF study material.
- Image.
- Admit card.
- App installer.

### Upload

Uploading means sending a file from your device to the internet.

Examples:

- Email attachment.
- Google Drive upload.
- Online admission form document upload.

### Safe Download Steps

1. Download from official/trusted websites.
2. Check file extension.
3. Avoid unknown `.exe` files.
4. Scan with antivirus.
5. Note save location.
6. Avoid pirated software and cracked games.

### Cookies

Cookies are small files saved by websites in the browser. They remember login sessions, language, cart items, and preferences. Advertising cookies can also track browsing behavior.

### Cache

Cache stores temporary copies of website images and files to load pages faster. Clear cache if a site shows old content or behaves incorrectly.

### Sessions

A session keeps you logged into a website temporarily. Always log out on shared devices.

### Incognito / Private Browsing

Private mode does not save local browsing history, but your ISP, school network, websites, or employer may still see activity. It is not full anonymity.

### Practice Exercise

Open browser settings and find options to clear browsing history, cache, and cookies.

## 15. Cybersecurity Awareness

### Phishing

Fake websites or emails that steal login details.

Example: fake bank login page claiming account will be blocked.

### Smishing

SMS-based phishing.

Example: `Your KYC expired. Click this link.`

### Vishing

Voice-call fraud.

Example: caller pretends to be bank officer and asks OTP.

### Malware

Harmful software that damages or steals data.

### Ransomware

Locks files and demands payment.

### Spyware

Secretly monitors user activity.

### Social Engineering

Manipulating people to reveal confidential information.

### Fake Job Scams

Fraudsters ask registration fees for fake jobs.

### UPI Frauds

Scammers ask users to enter UPI PIN to receive money. UPI PIN is needed to send money, not receive money.

### QR Code Scams

Scanning unknown QR codes may lead to payment requests or fake pages.

### Indian Safety Examples

- Fake courier KYC messages.
- Fake electricity bill disconnection links.
- Fake loan apps demanding contacts/photos access.
- Fake scholarship links asking processing fees.
- Fake customer care numbers found through random search results.

### Safe Browsing Checklist

- Use HTTPS for sensitive sites.
- Check domain carefully.
- Do not share OTP, PIN, passwords.
- Install updates.
- Use antivirus and firewall.
- Download from official sources.
- Report cybercrime at `cybercrime.gov.in`.

## 16. Misinformation and Deepfakes

### Fake News

False or misleading information spread as news.

### Deepfake Videos

AI-generated or altered videos that make people appear to say or do things they did not.

### AI-Generated Images

Images created by AI that may look realistic but are not real.

### Verification Techniques

- Check official sources.
- Search same news on trusted websites.
- Check date and context.
- Use reverse image search.
- Look for unnatural faces, hands, shadows, or audio mismatch.
- Avoid forwarding before verifying.

### Fact-Checking Websites

- PIB Fact Check
- Alt News
- Boom Live
- Factly

### Important Skill

Do not believe everything seen online.

## 1. Internet Basics

**Internet** = a global network (vishwa jaal) of millions of computers connected through cables, fibre, and wireless links.

**WWW (World Wide Web)** = a system of linked web pages on the internet, accessed through browsers using HTTP/HTTPS.

| Term | Meaning |
|------|---------|
| Website | Collection of related web pages on one server |
| Web page | Single HTML document shown in browser |
| Home page | First/main page of a website |
| Hyperlink | Clickable link to another page or site |
| Server | Computer that stores and sends web pages |
| Client | Your computer or phone that requests pages |

### How you connect to the internet

1. **ISP (Internet Service Provider)** — company that gives internet (Jio, Airtel, BSNL, local broadband)
2. **Modem** — converts signal from ISP line to digital data
3. **Router** — shares connection to many devices (often Wi-Fi)
4. **Wi-Fi** — wireless connection within home/office
5. **Mobile data (4G/5G)** — internet through SIM on phone

**IP Address** — unique number of a device on the internet (like house number).  
**Domain name** — easy name like `google.com` instead of remembering IP numbers.  
**DNS (Domain Name System)** — converts domain name to IP address.

---

## 2. Web Browser

A **browser** is software to open and view websites.

### Popular browsers (CCC exam)
- Google Chrome
- Microsoft Edge
- Mozilla Firefox
- Safari (Apple devices)

### Parts of browser window

| Part | Use |
|------|-----|
| Address bar | Type URL or search keywords |
| Tabs | Open many pages (Ctrl+T = new tab) |
| Back / Forward | Visit previous/next page |
| Refresh (F5) | Reload current page |
| Bookmarks / Favorites | Save site for quick access (Ctrl+D) |
| History (Ctrl+H) | List of pages you visited |
| Downloads (Ctrl+J) | Files saved from internet |
| Settings | Privacy, homepage, default search engine |

### Browser settings students should know
- Set **homepage** (page that opens on start)
- Clear **cache** and **cookies** when site behaves wrongly
- Enable **pop-up blocker** for safer browsing
- Turn on **safe browsing** warnings (Chrome/Edge)

---

## 3. URL — Uniform Resource Locator

**URL** = full web address of a page or file.

### Structure of a URL

```
https://www.example.gov.in:443/courses/index.html?page=1#top
└─┬─┘   └──────┬──────┘└┬┘ └──────┬──────┘ └────┬────┘ └┬┘
protocol    domain    port    path        query   fragment
```

| Part | Example | Meaning |
|------|---------|---------|
| Protocol | `https://` | Rules for connection |
| Domain | `www.example.gov.in` | Website name |
| Path | `/courses/index.html` | Location of page on server |
| Query | `?page=1` | Extra parameters |

### HTTP vs HTTPS

| Protocol | Security | Use |
|----------|----------|-----|
| HTTP | Not encrypted | General reading only |
| HTTPS | Encrypted (SSL/TLS) | Login, banking, payments |

**HTTPS** shows a **padlock icon** in address bar. Always use HTTPS for passwords and online payment.

### Common top-level domains (TLD)
- `.com` — commercial
- `.org` — organizations
- `.edu` — educational
- `.gov.in` — Indian government (official)
- `.nic.in` — National Informatics Centre (government)

---

## 4. ISP and Connection Types

**ISP** provides internet access for a fee (monthly plan / prepaid recharge).

| Connection type | Speed | Typical use |
|-----------------|-------|-------------|
| Broadband (DSL/Fibre) | High | Home, office |
| Mobile data | Medium–high | Phone, hotspot |
| Dial-up (old) | Very slow | Rare today |

### Data usage terms
- **Bandwidth** — amount of data that can flow per second (Mbps)
- **Data limit** — GB allowed per month in mobile plan
- **Hotspot** — share phone internet to laptop via Wi-Fi

## 5. Search Engines

**Search engine** finds web pages matching your keywords.

### Popular search engines
Google, Bing, DuckDuckGo, Yahoo

### Effective search tips (CCC useful)
- Use **specific keywords**: `NIELIT CCC syllabus 2025` not just `computer`
- **Quotes** for exact phrase: `"Digital India mission"`
- **site:** filter: `site:gov.in UMANG app`
- **filetype:** filter: `filetype:pdf CCC study material`
- Open results from **trusted sources** (.gov.in, .edu, official institute sites)

### Evaluating search results
- Check **date** of article — old news may be wrong
- Avoid clickbait titles
- Cross-check important facts on official website

## 6. Download and Upload

| Action | Direction | Example |
|--------|-----------|---------|
| Download | Internet → your device | Save PDF, image, app |
| Upload | Your device → internet | Attach file to email, cloud upload |

### Safe download steps
1. Download only from **official or trusted** websites
2. Check file **extension** (.pdf, .docx, .jpg) — avoid unknown .exe unless sure
3. Scan with **antivirus** after download
4. Note **save location** (Downloads folder)
5. Do not download **pirated software** or cracked games (virus risk)

### Upload uses
- Email attachments
- Google Drive / OneDrive
- Online form document upload (admission, job application)

---

## 7. Cookies, Cache, and Sessions

### Cookies
Small text files stored by browser on your computer. Websites use cookies to:
- Remember login (session cookie)
- Save preferences (language, theme)
- Track usage (advertising cookies)

**Risk:** Third-party cookies can track browsing. Clear cookies or use private/incognito mode on shared PC.

### Cache
Temporary copies of images and pages stored locally for **faster loading**.  
If a website shows old content, clear cache and refresh.

### Incognito / Private browsing
- Does not save history on local PC (still visible to ISP/school network admin)
- Useful on **cyber café** or library computer
- Always **log out** after banking/email on shared PC

---

## 8. Internet Safety While Browsing

### Common online risks
| Risk | What happens | Protection |
|------|--------------|------------|
| Phishing site | Fake login page steals password | Check URL, padlock, official domain |
| Malware download | Virus in fake software | Official app stores only |
| Fake offers | "Free iPhone" scam links | Do not click suspicious ads |
| Public Wi-Fi sniffing | Data intercepted | Avoid banking on open Wi-Fi |

### Safe browsing checklist
- Use **HTTPS** for sensitive sites
- Keep **browser and Windows updated**
- Install **antivirus** and enable firewall
- Do not share **OTP, PIN, password** on call or chat
- Verify **.gov.in** for government schemes
- Report fraud at **cybercrime.gov.in**

### Parental and student discipline
- Limit screen time (samay prabandhan)
- Do not meet strangers from chat without guardian
- Think before posting personal photos (digital footprint)
